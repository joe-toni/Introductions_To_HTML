<figcaption>:

A <figcaption> is a tag usually used within a figure parent along with an <img> tag to indicate that the text contained is a caption for the preceding image. It has no special renduring but can be styled by name in css and is expected to either be the first or last child within the parent <figure> tag.

A <blockquote> is used to indicate that the text contained within is an extended quote. It is typically used along with a <cite> and <p> with the cite tag providing a source for the quote and a the paragraph tag actually containing the disired text. When rendured basically the blockquote tags will add indentation to the paragraph in order to indicate a quotation but this can be adjusted through styling the margin.

The Description Details tag or <dd> is used within list elements to provide some extra description or definition to the corresponding list item that precedded it.

The <dl> tag is used to define a description list. It doesn't render much different that a regular list but is considered semantically correct when intending to create a list elment with discriptive content.

This element is called the description term or <dt> and is meant to be used within a description list and is expected to be followed by a description details <dd> unless multiple terms are to be defined by the same description. In this case multiple terms can be used in a row but must still end in a description.

Headers are actually a collection of tags used to create headings for areas of our site with a few options that aside from controling size can also serve the semantic purpose of labeling by order of improtance with h1 being both the biggest and the most important.

The horizontal rule element does not itself contain text but is usually used between bodies of text or sections to better indicate a separtation of content.

The &lsli> is known as the list item element and it is meant to be used in either ordered, unordered lists, or menus. It will render with bullet points if used in a menu or unordered list while renduring with accending numbers in an ordered list. Note that while it can technically be used in a description list it would be more correct for the author to specify a description term in such a case.

An ordered list elment or <ol> is meant to be used for lists where the order of appearance is important hence the list items renduring with a numreical value at the begining of each item.

The paragraph tag or <p> is used to contain any grouping of related content although typically thought of as meant to contain blocks of text.

<pre> or the preformated text element is similar to the paragraph tag focused on containing blocks of text, however while a p element will ignore extra spaces a preformated element will render the text exactly as writen.

An unorderd list item or <ul> is an elment similar to the <ol> or the <dl> elements how ever this element is meant to be used on lists where the order appearance is unimportant hence it renduring with bullet points as apposed to numerical values.
